# Architecture Overview

Languages: [English](ARCHITECTURE_OVERVIEW.md) | **繁體中文**

本文件只描述適合公開討論的高層架構，不包含 private deploy 細節或敏感 bridge 設定。

## 核心概念

PHINIX 不是單純聊天機器人，也不是單純工具代理。

它更接近：

- 本地主權 runtime
- 具身 companion 主腦
- 未來人形機器人的 cognition layer

## 建議的三層架構

### 1. 即時處理層

職責：

- 接收即時輸入
- 產生低延遲回應
- 驅動當下互動

典型通道：

- 語音
- 文字
- HUD / AR 輸出

### 2. 緩衝層

職責：

- 保留事件與對話上下文
- 產生狀態快照
- 管理待處理問題

這一層最重要的不是智慧，而是：

- 承接
- 結構化
- 保留可回看性

### 3. 思考層

職責：

- 背景推理
- 卡關回看
- 長時主題追蹤
- 主動輸出候選結果

這一層不應取代即時層，而應與即時層並行。

## Stuck issue queue

PHINIX 的一個核心差異，是把「卡關」當成可持續追蹤的正式狀態，而不是一次性失敗。

這讓系統可以：

- 記住失敗發生在哪裡
- 記住已經試過什麼
- 之後在新線索出現時再回頭處理
- 在適當時機主動提醒使用者

## Proactive companion 行為

主動性不應等於持續打擾。

較健康的流程是：

1. 背景產生新結論
2. 準備低打擾提醒
3. 由使用者選擇是否展開

這才比較接近 companion，而不是噪音來源。

## 對未來機器人的意義

如果未來接上人形機器人，PHINIX 比較可能扮演：

- 記憶
- 問題追蹤
- 主動推理
- 治理
- 長時一致性

而不是：

- 低階控制器
- 硬即時運動控制
- 單純 SDK wrapper

因此可以把 PHINIX 視為能夠跨裝置、跨身體延續的可攜式主腦核心。
